# LearnNanoXRP

**Learn NanoXRP** — A self-paced, SCORM-compatible learning site for the [XRP (Experiential Robotics Platform)](https://www.sparkfun.com/products/22727) robot by SparkFun / WPI.

🌐 **Live Site:** `https://saintsampo.github.io/LearnNanoXRP`

---

## What's Inside

Six lessons, each with guided slides, interactive content, and a quiz:

| # | Lesson | Key Concept |
|---|--------|-------------|
| 1 | 🔧 Assembling the Robot | Parts, safety, step-by-step build |
| 2 | 💻 The Web Editor & Blockly | XRP Web editor, Blockly drag-drop programming |
| 3 | 🚗 Making the Robot Drive | **Blocking vs non-blocking** drive commands |
| 4 | 〰️ Following a Line | Reflectance sensors, feedback loop |
| 5 | 📡 Sonar Sensor | Echolocation, stop-before-wall |
| 6 | 🎮 Wireless Gamepad | Arcade drive, button actions |

Progress is saved in the **browser's local storage** — students can close the tab and pick up where they left off.

---

## Using the GitHub Pages Site

1. Open `https://jrw4561.github.io/LearnNanoXRP` in Chrome or Edge.
2. Click **Start Learning** or a lesson from the left sidebar.
3. Complete all slides and pass the quiz to unlock the next lesson.
4. Progress bar at the top shows overall completion.

## Downloading SCORM Packages (for Canvas / Moodle / any LMS)

Pre-built SCORM 1.2 ZIP packages are in the [`scorm-packages/`](scorm-packages/) folder:

| File | Lesson |
|------|--------|
| `lesson1-assembly.zip` | Assembling the Robot |
| `lesson2-web-editor.zip` | The Web Editor & Blockly |
| `lesson3-driving.zip` | Making the Robot Drive |
| `lesson4-line-follow.zip` | Following a Line |
| `lesson5-sonar.zip` | Sonar Sensor |
| `lesson6-gamepad.zip` | Wireless Gamepad |

### Uploading to Canvas
1. Go to your Canvas course → **Modules** → **+ Add Item**
2. Choose **External Learning Tools** → **SCORM**  
   *(or use the Files area if your school uses a SCORM player)*
3. Upload the ZIP file.

### Uploading to Moodle
1. Turn editing on → **Add an activity or resource**
2. Choose **SCORM package** → Upload the ZIP.
3. Set completion criteria to **Score ≥ 75%**.

## Adding Real Photos (replacing placeholder images)

Every image slot currently has an **SVG placeholder** that describes exactly what photo to take. Here's how to swap in real photos:

1. **Find the placeholder** in `lessons/<lesson>/images/` — each SVG includes a comment describing what the photo should show.
2. **Take or find a photo** matching the description.
3. **Save it** as a `.png`, `.jpg`, or `.webp` file with the same base name (e.g., `01-completed-robot.png`).
4. **Update the `<img src="">`** in the lesson HTML from `.svg` to your new file extension.

Example — in `lessons/lesson1-assembly/index.html`:
```html
<!-- Before (placeholder) -->
<img src="images/01-completed-robot.svg" alt="..." />

<!-- After (real photo) -->
<img src="images/01-completed-robot.png" alt="..." />
```

### Regenerate placeholders
If you need to recreate the placeholder images from scratch:
```bash
node generate-placeholders.js
```

## Development

```bash
# Install dev tools
npm install

# Regenerate all placeholder images
npm run generate-images

# Build SCORM packages
npm run package-scorm

# Do both
npm run build
```

## GitHub Pages Setup

1. Go to your repo **Settings → Pages**.
2. Set **Source** to `GitHub Actions`.
3. Push any commit to `main` — the workflow builds + deploys automatically.
