🔍 Meet the Reflectance Sensors

Your robot has a line follower board on the bottom with two tiny sensors: a left sensor and a right sensor.

Bottom of the robot — the two reflectance sensors are at the front
💡
What are reflectance sensors?

Each sensor has a tiny infrared (IR) light that shines down at the floor, and a receiver that measures how much light bounces back.

  • White surface → bounces lots of light back → sensor reads low value (near 0)
  • Black line → absorbs the light → sensor reads high value (near 1)
How IR reflection works: black absorbs light, white reflects it

📊 Reading Sensor Values

In Blockly, you can read the sensor values using blocks in the Sensors → Reflectance category.

The left and right reflectance sensor blocks

What the values mean:

Sensor Reading What the sensor sees Robot action
Low (0.0 – 0.4) White floor (off the line) Steer back toward line
High (0.5 – 1.0) Black tape (on the line!) On track!
💡
The exact threshold depends on your floor

Different floors have different brightness. You'll need to test what value works best — try using 0.5 as your dividing line between white and black.

🗺️ Setting Up Your Track

Before you can test line following, you need to create a track on the floor!

Use black electrical tape or masking tape on a white or light-colored surface
1

Choose your surface

You need a light-colored flat surface — a white poster board or a light-colored floor works great.

2

Apply the tape

Use black electrical tape (about 2 cm wide) to create a track. Start simple — an oval or a big letter O is great for beginners!

3

Smooth out the tape

Press the tape down firmly so it sticks flat. Bumpy or wrinkled tape can confuse the sensor.

4

Avoid sharp corners

Keep your track curves gentle — the robot needs time to react! A 90-degree corner will be very hard to follow.

🧠 The Line-Following Logic

Here's how the robot "thinks" about following a line. It uses a strategy called bang-bang control (an on-or-off approach):

🤔
The idea:
  • If the right sensor sees the black line → steer right (you've drifted left)
  • If the left sensor sees the black line → steer left (you've drifted right)
  • If neither sensor sees the line → drive straight
Steering logic: sensors detect drift and correct the robot's path
🤖
This is a feedback loop!

The robot senses its position, compares it to where it should be, and makes a correction. This is the same idea used in airplane autopilots, thermostats, and self-driving cars!

🧱 Building the Code in Blockly

Here's the Blockly program for basic line following. This is a non-blocking approach because we need to constantly check the sensors while driving.

Your complete line-following program in Blockly
🔁 repeat forever
if 📡 right sensor > 0.5
⚡ set effort left 0.4 right 0.1 (steer right)
else if 📡 left sensor > 0.5
⚡ set effort left 0.1 right 0.4 (steer left)
else
⚡ set effort left 0.3 right 0.3 (go straight)
⏱ wait 0.02 seconds

The wait 0.02 at the end keeps the loop running at a steady rate — about 50 times per second!

🔧 Testing and Tuning

Run your line-following program and watch the robot go! If it doesn't work perfectly the first time, that's okay — tuning is part of engineering.

Place the robot on the track and run your program!

Common problems and fixes:

🔧

Robot goes off the track

The threshold (0.5) might be wrong for your floor. Try reading the sensor values and printing them to the terminal — then use a value between what you see for white and black.

🔧

Robot wiggles too much

Reduce the difference between the turning speeds. Instead of 0.4/0.1, try 0.35/0.2.

🔧

Robot is too slow or too fast

Adjust the base speed (the 0.3 in the "go straight" case). Too fast and the robot can't react in time; too slow and it barely moves.

🔧

Robot gets confused at corners

Make your track corners more gradual, or slow down the robot's base speed for tighter turns.

💡
Figure-8 tips
  • Make each loop large — at least 60 cm across
  • The crossing point in the middle should be a smooth X, not a sharp corner
  • Start the robot near the center and face it toward one loop

Advanced challenge
