🎮 Drive Your Robot Like a Video Game!

In this final lesson, you'll learn how to control your NanoXRP robot with a gamepad controller — just like a video game!

Control your robot with a real gamepad controller!
🤔
How does it work?

A gamepad (like an Xbox or PlayStation controller) plugs into your computer. The XRP Web Editor reads the gamepad's joystick positions and button presses, then sends that information to your robot through the USB or Bluetooth connection.

You'll learn:

  • How to connect a gamepad to your computer
  • The gamepad layout — joysticks, buttons, and the D-pad
  • How to use Gamepad blocks in Blockly
  • How to program arcade drive (one joystick controls speed & direction)
  • How to use buttons for extra actions

🎮 Connecting Your Gamepad

First, let's connect the gamepad to your computer and get the XRP Web Editor to recognize it.

Connect the gamepad and look for the controller icon in the web editor
1

Plug in the gamepad

Connect your gamepad to your computer using its USB cable (or receiver dongle for wireless gamepads). Windows should automatically install the driver.

2

Press any button on the gamepad

The browser needs you to press a button on the gamepad before it will recognize it. Press any button (A, B, etc.) once.

3

Look for the gamepad indicator

In the XRP Web Editor toolbar, you'll see a gamepad icon appear when the controller is detected. The icon will light up when you move the joysticks.

4

Connect your robot too

Make sure your robot is connected to the editor via USB or Bluetooth — you need both the gamepad and robot connected at the same time.

🕹️ Gamepad Layout

Let's learn the different parts of the gamepad and what their names are in the XRP system.

The main parts of a standard gamepad controller
Part XRP Name Values
Left joystick (left/right) LEFT_X -1.0 (left) to +1.0 (right)
Left joystick (up/down) LEFT_Y -1.0 (up) to +1.0 (down)
Right joystick (left/right) RIGHT_X -1.0 (left) to +1.0 (right)
A, B, X, Y buttons BTN_A, BTN_B, etc. 0 (released) or 1 (pressed)
Left/Right Bumpers BTN_L, BTN_R 0 or 1

🧱 Gamepad Blocks in Blockly

The Gamepad category (orange) has two blocks for reading gamepad input.

The two Gamepad blocks in Blockly
1

get gamepad value [axis]

Returns the current position of a joystick axis. Choose which axis from the dropdown (LEFT_X, LEFT_Y, RIGHT_X, RIGHT_Y, etc.).

🎮 get gamepad value LEFT_Y

Returns a number from -1.0 to 1.0

2

is button pressed [button]

Returns TRUE if the selected button is currently pressed, FALSE if not.

🎮 is button pressed BTN_A

Returns True or False

🕹️ Arcade Drive

Arcade drive uses ONE joystick to control both speed and steering — like most racing video games. The XRP has a built-in arcade() block for this!

🤔
How arcade drive works
  • Push the joystick up → robot goes forward
  • Push the joystick down → robot goes backward
  • Push the joystick left → robot turns left
  • Push the joystick right → robot turns right
  • Push it diagonally → go forward AND turn at the same time!
One joystick = full robot control!
⚠️
The Y-axis is flipped!

On most gamepads, pushing the joystick UP gives a NEGATIVE Y value (-1.0). So when you code it, you'll need to negate the Y value: use -(LEFT_Y) so that up = positive (forward).

🧱 Building the Arcade Drive Program

Here's the complete arcade drive program using the left joystick. The robot continuously reads the joystick and updates its motors.

The complete gamepad arcade drive program
# Run forever, updating every loop
🔁 repeat forever
🚗 arcade drive
straight: - 🎮 get gamepad value LEFT_Y
turn: 🎮 get gamepad value LEFT_X
⏱ wait 0.02
💡
Why negate LEFT_Y?

Most controllers report up as -1 and down as +1. Since we want up = forward = positive, we flip the sign with a "negate" block (the minus sign block in Math category).

🏆
Ideas for button actions
  • B button: Honk (blink LED rapidly)
  • X button: Spin in place
  • Y button: Emergency stop
  • Left bumper: Slow speed mode
  • Right bumper: Turbo speed mode
