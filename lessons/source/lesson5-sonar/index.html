👁️ The Sonar Sensor on Your Robot

The ultrasonic sensor is the part at the front of your robot that looks like two round eyes. It's called a rangefinder.

The two "eyes" on the front of the robot are the sonar sensor

The two circles are:

📢 Transmitter

Sends out a very short burst of ultrasonic sound (too high-pitched for humans to hear).

👂 Receiver

Listens for the echo of that sound bouncing back off objects.

💡
How it calculates distance

The sensor measures how long it takes for the echo to return. Sound travels at about 343 meters per second, so the robot's computer can calculate:
Distance = (time × speed of sound) ÷ 2
Dividing by 2 because the sound travels there AND back!

🔊 How Sonar Measures Distance

Let's look at exactly what happens each time the sonar takes a measurement.

Sound pulse goes out → bounces off object → echo returns → distance calculated!
1

The sensor sends a pulse

The transmitter sends a quick "ping" of ultrasonic sound — 8 pulses at 40,000 Hz. You can't hear it!

2

Sound travels through air

The sound moves away from the robot at the speed of sound — about 343 m/s (the same as a thunderclap!).

3

Echo bounces back

When the sound hits a solid object, it bounces back like an echo in a cave.

4

Distance is calculated

The sensor measures how long the round trip took and converts that to centimeters. The block returns this value to your program.

🧱 The Distance Block in Blockly

Reading the sonar sensor in Blockly is super simple! There's one block for it in the Sensors → Distance category.

The distance block — it returns the distance in centimeters
📡 get sonar distance (cm)

This block returns a number in centimeters. You can use it anywhere a number is expected — in an if-statement, stored in a variable, or printed to the terminal.

Testing the sensor — try this first:

🔁 repeat forever
📝 print 📡 get sonar distance (cm)
⏱ wait 0.1

Run this and watch the Terminal! Move your hand in front of the sensor and see the numbers change. When nothing is in range, it might show a very large number (like 999 or similar).

🛑 Stop Before Hitting a Wall

Now let's write the full program: drive forward and stop when something is within 20 cm.

The complete stop-before-wall program
# Start driving forward
⚡ set effort left 0.5 right 0.5

# Keep going until something is close
🔁 repeat until 📡 distance < 20
⏱ wait 0.05

# Stop!
⏹ stop motors
💡
Why "repeat UNTIL"?

"Repeat until" runs the loop as long as the condition is NOT true, then exits when it IS true. So the loop keeps repeating while the distance is 20 cm or more, and exits as soon as the distance drops below 20 cm.

🧪 Testing Your Stop-at-Wall Program

A stack of books makes a great test wall!

Set up your "wall"

2

🐢 Challenge: Slow Down Gradually

Right now the robot drives at full speed and then abruptly stops. Can we make it slow down first, like a real car?

A gradual slowdown is smoother and more realistic!

Here's the idea using three speed zones:

🔁 repeat forever
if 📡 distance < 15
⏹ stop motors
else if 📡 distance < 30
⚡ set effort left 0.2 right 0.2 (slow)
else
⚡ set effort left 0.5 right 0.5 (full speed)
⏱ wait 0.05

Now the robot: drives fast when far away → slows down at 30 cm → stops at 15 cm!

💡 Sonar Sensor Tips

The sonar sensor is powerful but has some quirks. Here's what to know:

✅

Works best on flat, hard surfaces

Books, walls, boxes, and hard furniture give clean readings. Soft objects like pillows absorb sound and give strange results.

✅

Range: about 2 cm to 400 cm

The sensor struggles with objects too close (under 2 cm) or too far away (over 4 meters). For classroom use, 5–200 cm works great.

⚠️

Narrow detection cone

The sonar only "sees" in a narrow cone directly ahead. An object off to the side might not be detected — that's why the sensor needs to point directly at what it's measuring.

⚠️

Watch for false readings

If the sensor returns a very large or very small number unexpectedly, it might be bouncing off something behind you or the floor. Add a check: if distance > 300 or distance < 2, ignore the reading.
