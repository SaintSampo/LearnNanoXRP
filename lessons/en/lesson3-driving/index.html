🧱 DriveTrain Blocks Overview

All the driving commands are in the DriveTrain category (the rust-orange blocks). Here's what each one does:

The DriveTrain category has all your driving commands
Block Name What it does Type
straight(dist, effort) Drive forward or backward a set distance BLOCKING
turn(degrees, effort) Rotate left or right a set number of degrees BLOCKING
set_effort(left, right) Set motor power immediately (-1 to 1) NON-BLOCKING
set_speed(left, right) Set motor speed in RPM NON-BLOCKING
stop() Stop both motors immediately NON-BLOCKING

⏸️ What is a BLOCKING Command?

A blocking command makes the program pause and wait until the command is completely finished before moving on to the next line.

🤔
Think of it like a crosswalk signal

When you press the button to cross, you wait at the curb until the light turns green. You don't cross yet — you're "blocked" until it's done. That's exactly what a blocking command does!

Blocking: Step 1 must FINISH before Step 2 can start

The two blocking drive commands are:

📏

straight(distance, effort)

Drives the robot exactly that far and stops. The program waits until the robot has traveled that whole distance before running the next block.

distance = centimeters (20 = 20 cm forward, -20 = backward)

effort = motor power (0.0 to 1.0 — try 0.5 to start!)

↩️

turn(degrees, effort)

Turns the robot exactly that many degrees and stops. 90 degrees = a quarter turn. 180 degrees = a half turn (U-turn).

degrees = positive = turn right, negative = turn left

Why this works perfectly

Because straight() and turn() are blocking, the robot finishes driving 30 cm completely before turning. If they weren't blocking, the robot would try to drive and turn at the same time — total chaos!

▶️ What is a NON-BLOCKING Command?

A non-blocking command starts the action and immediately moves on to the next line of code — it doesn't wait!

🤔
Think of it like a microwave

You press Start and then walk away to do something else. The microwave is running, but you're already doing the next thing! That's non-blocking.

Non-blocking: the code keeps running while the motors are going!

Non-blocking drive commands:

⚡

set_effort(left, right)

Sets both motors to a power level from -1.0 to 1.0. Positive = forward, negative = backward. The motors keep spinning until you tell them to stop!

set_effort(0.5, 0.5) = both motors at 50% forward

set_effort(-0.5, -0.5) = both motors at 50% backward

set_effort(0.5, -0.5) = spin in place (left forward, right backward)

⚡

stop()

Stops both motors right now. Use this after set_effort() to stop the robot at the right time.

⚡ Non-Blocking in Action

Non-blocking commands are used when you decide when to stop — often with a sensor reading or a timer.

Example: Drive for 2 seconds, then stop

⚡ set effort left 0.5 right 0.5

⏱ wait 2 seconds

⏹ stop motors
💡
What happens here?
  1. set_effort(0.5, 0.5) starts both motors — code moves immediately to the next block
  2. wait(2) pauses the code for 2 seconds (but the motors keep going!)
  3. stop() turns both motors off

Example: Drive and check sensor at the same time

⚡ set effort left 0.5 right 0.5

🔁 repeat until distance < 20 cm

⏱ wait 0.05 seconds

⏹ stop motors

The robot drives forward and keeps checking the sonar sensor. As soon as something is within 20 cm, it stops!

🤔 When to Use Each Type

Choosing the right type of command is like choosing the right tool for the job. Here's a guide:

✅ Use BLOCKING when...

  • You want to drive an exact distance
  • You want to turn an exact number of degrees
  • Steps must happen one at a time in order
  • Example: drive in a square, maze navigation

✅ Use NON-BLOCKING when...

  • You want the robot to drive while checking a sensor
  • You want to control speed gradually
  • You're using a gamepad (continuous control)
  • Example: stop-at-wall, line following, gamepad control
⚠️
Common mistake with non-blocking!

If you forget to call stop() after set_effort(), the robot will keep going forever until the batteries die or it crashes! Always remember to stop.

📝 Summary

Let's review what you've learned about making the robot drive!

⏸️ BLOCKING

straight(dist, effort)
Drive exact distance, wait until done.

turn(deg, effort)
Turn exact degrees, wait until done.

Best for: step-by-step paths

▶️ NON-BLOCKING

set_effort(L, R)
Set motor power, keep going.

stop()
Stop motors right now.

Best for: sensor-based stopping, gradual control
